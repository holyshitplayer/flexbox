Properties

Контейнер

display

Это свойство определяет тип отображения элемента: flex-контейнер, inline или block, в зависимости от заданного значения. Он включает flex-контекст для всех своих прямых дочерних элементов.

.container { display: flex; /* or inline-flex */ }

flex-direction

Это свойство определяет основную ось, тем самым определяя направление размещения flex-элементов в flex-контейнере. Flexbox — это (помимо опциональной обертки) концепция макета с одним направлением. Думайте о гибких элементах как о расположении в первую очередь либо в горизонтальных рядах, либо в вертикальных столбцах.

flex-direction: row
flex-direction: row-reverse
flex-direction: column
flex-direction: column-reverse
.container { flex-direction: row | row-reverse | column | column-reverse; }

  • row (по умолчанию): слева направо в LTR (Left to Right); справа налево в RTL (Right to Left)

  • row-reverse: справа налево в LTR (Left to Right); слева направо в RTL (Right to Left)

  • column: то же, что row, но сверху вниз

  • column-reverse: то же, что и row-reverse, но снизу вверх

flex-wrap

По умолчанию все flex-элементы будут пытаться разместиться в одной строке. С помощью этого свойства Вы можете изменить это и разрешить элементам переноситься на следующую строку по мере необходимости.

flex-wrap: wrap
flex-wrap: wrap-reverse
.container { flex-wrap: nowrap | wrap | wrap-reverse; }

  • nowrap (по умолчанию): все flex-элементы будут в одной строке.

  • wrap: flex-элементы будут переноситься на следующую строку сверху вниз.

  • wrap-reverse: flex-элементы будут переноситься на следующую строку снизу вверх.

flex-flow

Это сокращение для свойств flex-direction и flex-wrap, которые вместе определяют главную и поперечную оси flex-контейнера. Значение по умолчанию — row nowrap.

.container { flex-flow: column wrap; }

justify-content

Это свойство определяет способ выравнивания вдоль главной оси. Это помогает распределять дополнительное свободное пространство, когда либо все flex-элементы в строке не растяжимые, либо растяжимые но достигли своего максимального размера. Свойство также осуществляет некоторый контроль над выравниванием элементов, когда они выходят за пределы строки.

justify-content: flex-start
justify-content: flex-end
justify-content: center
justify-content: space-between
justify-content: space-around
justify-content: space-evenly
.container { justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly | start | end | left | right ... + safe | unsafe; }

  • flex-start (по умолчанию): элементы располагаются от начала основной оси учитывая flex-direction.

  • flex-end: элементы располагаются к концу основной оси учитывая flex-direction.

  • start: элементы располагаются от начала основной оси не учитывая flex-direction.

  • end: элементы располагаются к концу основной оси не учитывая flex-direction.

  • left: элементы располагаются к левому краю контейнера. Если основная ось не горизонтальная, тогда это значение ведет себя как start.

  • right: элементы располагаются к правому краю контейнера. Если основная ось не горизонтальная, тогда это значение ведет себя как start.

  • center: элементы располагаются по центру строки.

  • space-between: элементы равномерно распределяются в строке; расстояния между каждой парой соседних элементов равны; первый элемент прижат к началу контейнера, последний элемент прижат к концу контейнера.

  • space-around: элементы равномерно распределяются в строке с равным пространством вокруг них. Обратите внимание, что визуально пространства не равны, так как все предметы имеют одинаковое пространство с обеих сторон. Первый элемент будет иметь одну единицу пространства от края контейнера, но будет иметь две единицы пространства между следующим элементом, потому что следующий элемент имеет свой собственный интервал.

  • space-evenly: элементы распределяются так, чтобы расстояние между любыми двумя элементами (и расстояние до краев) было одинаковым.

Обратите внимание, что поддержка браузерами этих значений имеет нюансы. Например, значение space-between никогда не поддерживалось в некоторых версиях Edge, а start/end/left/right нет в некоторых версиях Chrome. Наиболее безопасными для использования значениями являются flex-start, flex-end и center.

Есть также два дополнительных ключевых слова, которые вы можете сочетать с этими значениями: safe и unsafe. Использование safe гарантирует, что, какой бы тип позиционирования не использовался, элемент не будет отображаться за пределами экрана (например, сверху), чтобы появлялась полоса прокрутки в контейнере.

align-items

Это свойство определяет способ расположения элементов вдоль поперечной оси на текущей строке.

align-items: flex-start
align-items: flex-end
align-items: center
align-items: stretch
align-items: baseline
.container { align-items: stretch | flex-start | flex-end | center | baseline | first baseline | last baseline | start | end | self-start | self-end + ... safe | unsafe; }

  • stretch (по умолчанию): растянуть, чтобы заполнить контейнер (по-прежнему соблюдается минимальную/максимальную ширина и высота).

  • flex-start / start / self-start: элементы прижаты к началу поперечной оси. Между ними небольшая разница и заключается в соблюдении текущего значения flex-direction.

  • flex-end / end / self-end: элементы прижаты к концу поперечной оси. Разница снова заключается лишь в соблюдении текущего значения flex-direction.

  • center: элементы центрированы по поперечной оси.

  • baseline: элементы выравниваются таким образом, что их базовые линии совпадают. Элемент с наибольшим расстоянием между его началом поперечного края и его базовой линией выравнивается с началом поперечного края строки.

Ключевые слова-модификаторы safe и unsafe можно использовать в сочетании со всеми остальными ключевыми словами (хотя обратите внимание на поддержку браузерами) и помочь Вам предотвратить выравнивание элементов таким образом, что содержимое становится недоступным.

align-content

Это свойство определяет способ выравнивания строк внутри flex-контейнера, когда на поперечной оси есть дополнительное пространство, аналогично тому, как justify-content выравнивает отдельные элементы внутри основной оси.

Примечание: Это свойство действует только для многострочных flex-контейнеров (где для flex-wrap установлено значение wrap или wrap-reverse). Для однострочного flex-контейнера (т.е. где для flex-wrap установлено значение по умолчанию) свойство align-content работать не будет.

align-content: flex-start
align-content: flex-end
align-content: center
align-content: space-between
align-content: space-around
align-content: space-evenly
.container { align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch | start | end | baseline | first baseline | last baseline + ... safe | unsafe; }

  • normal (по умолчанию): элементы располагаются так, как если бы значение не было установлено.

  • flex-start / start: элементы располагаются от начала поперечной оси, учитывая flex-direction для flex-start и не учитывая flex-direction для start.

  • flex-end / end: элементы располагаются к концу поперечной оси, учитывая flex-direction для flex-end и не учитывая flex-direction для end.

  • center: элементы по центру контейнера.

  • space-between: строки равномерно распределяются в контейнере; расстояния между каждой парой соседних строк равны; первая строка прижата к началу контейнера, последняя строка прижата к концу контейнера.

  • space-around: строки равномерно распределяются в контейнере с одинаковым пространством вокруг каждой строки.

  • space-evenly: строки распределяются так, чтобы расстояние между любыми двумя строками (и расстояние до краев) было одинаковым.

  • stretch: линии растягиваются так, чтобы занять пустое пространство в контейнере.

Ключевые слова-модификаторы safe и unsafe можно использовать в сочетании со всеми остальными ключевыми словами (хотя обратите внимание на поддержку браузерами) и помочь Вам предотвратить выравнивание элементов таким образом, что содержимое становится недоступным.

gap, row-gap, column-gap

Свойство gap явно задаёт отступы между flex-элементами. Это расстояние применяется только между элементами и не применяется по краям элементов.

gap: 10px
gap: 30px
gap: 10px 30px
column-gap: 20px
row-gap: 20px
.container { display: flex; gap: 10px; gap: 10px 20px; /* row-gap column gap */ row-gap: 10px; column-gap: 20px; }

Поведение этого свойства можно рассматривать как минимальный отступ между элементами. Отступ между элементами может стать больше (например, если применить justify-content: space-between;), но не может стать меньше установленого отступа.

Отступ задаётся не только между элементами, но и между строками. В зависимости от свойства можно отдельно указать отступ между элементами и между строками.
